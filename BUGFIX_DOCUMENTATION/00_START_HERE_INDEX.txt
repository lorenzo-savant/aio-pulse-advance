🔴 THE PROBLEM (In One Minute)
═══════════════════════════════════════════════════════════════════════════════

Your AIO Pulse dashboard shows 500 errors when loading data:

ERROR: GET /api/scans 500 (Internal Server Error)
ERROR: POST /api/prompts 500 (Internal Server Error)

ROOT CAUSE:
───────────
Supabase environment variables are missing in your Vercel production deployment.

When Vercel tries to start your app without these variables:
1. Database client initialization fails
2. API routes crash when trying to use the database
3. Returns 500 error instead of a helpful message
4. Dashboard can't load scan history

PLUS:
─────
Browser console also shows Permissions-Policy warnings (annoying but non-critical)

═══════════════════════════════════════════════════════════════════════════════

✅ THE SOLUTION (In One Minute)
═══════════════════════════════════════════════════════════════════════════════

You need to add 3 Supabase credentials to your Vercel production environment:

STEP 1: Get the credentials from Supabase
──────────────────────────────────────────
Go to: https://supabase.com
Login → Your Project → Settings → API

Copy these 3 values:
  • Project URL → for NEXT_PUBLIC_SUPABASE_URL
  • Anon Key → for NEXT_PUBLIC_SUPABASE_ANON_KEY
  • Service Role Key → for SUPABASE_SERVICE_KEY

STEP 2: Add them to Vercel
──────────────────────────
Go to: https://vercel.com/dashboard
Select project → Settings → Environment Variables

Add 3 new variables:
  1. Name: NEXT_PUBLIC_SUPABASE_URL
     Value: (paste your Project URL)
  2. Name: NEXT_PUBLIC_SUPABASE_ANON_KEY
     Value: (paste your Anon Key)
  3. Name: SUPABASE_SERVICE_KEY
     Value: (paste your Service Role Key)

STEP 3: Redeploy
────────────────
Go to: Deployments → Latest → Redeploy

STEP 4: Verify
──────────────
Visit your app → Open DevTools (F12) → Check Console
Look for: No more 500 errors! ✅

TOTAL TIME: ~15 minutes
DIFFICULTY: Easy (just configuration, no coding needed)
